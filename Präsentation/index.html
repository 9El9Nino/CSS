Basics

  • CSS wurde entworfen um Darstellungsvorgaben von den Inhalten zu trennen
  • HTML = inhaltliche Gliederung und Bedeutung
  • CSS = Darstellung der Inhalte

Rules

  • Eigenschaften sind innerhalb von Regelsätzen organisiert
  • Ein Regelsatz besteht aus einem/mehreren Selektoren und einem begrenzten Bereich in dem ein Wert zugewiesen ist

Selektor {Eigenschaft:Wert;}

body {background-color:red;}

Selectors

  • Über Selektoren werden bestimmte Elemente in einem HTML-Dokument ausgewählt
  • Auf diese Elemente können bestimmte Eigenschaften angewandt werden
  • Z.B. body {background-color:red;}
  • In diesem Beispiel wird vom body der HTML-Datei der Hintergrund rot

Declaration Blocks

  • Jede Deklaration besteht aus einer Eigenschaft und einem Wert
  • Ein Deklarationsblock umfasst den gesamten Teil einer Regel hinter dem Selektor
  • Z.B. body {background-color:red;}
  • Man kann auch leere Deklarationen nutzen

Inheritance

  • Vererbung macht ein Stylesheet übersichtlicher
  • Ein gutes Beispiel wäre hierfür die Vererbung der Schrift
  • body {font-family: Arial, Helvetica, sans-serif;}
  • Dadurch wird es einfacher einzelne Styles für alle Deklarationsblöcke zu übernehmen

Specificity

  • Sobald Stylesheets zu lang werden, werden diese schnell unübersichtlich
  • Das führt dazu, dass manche Elemente mit mehreren CSS-Regeln belegt werden
  • Durch ein Punktestystem bestimmt ein Browser, welche Eigenschaftswerte die relevantesten für ein Element sind

Specificity

  • Dies basiert auf Matchingregeln, welche durch die verschiedenen Selektoren gebildet werden
  • Die Anzahl jedes Selektortyps wird berechnet
  • Bei Spezifitätgleichheit wird die letzte im CSS gefundene Deklaration für das Element angewannt
